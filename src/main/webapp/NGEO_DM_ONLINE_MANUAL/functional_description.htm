FUNCTIONAL DESCRIPTION

The DMngEO DownloadManager, although based on standard data download functionalities, is meant to be used in full integration with the ngEO infrastructure.
 
From the user perspective, this means that the user shall have access to the ngEO Web Client (see [RD.1] for a general description of the ngEO architecture).
 
From the Web Client, the user searches for data and, possibly starts the download of the data. For the data download there are basically 2 options:
 
  • Direct local download, exploiting the browser built in download manager
  • Download using the Download Manager specifically designed for the ngEO
 
While in many simple data access the user can simply exploit the browser download manager, the usage of the DMngEO DownloadManager offers the following additional functionalities, for example:
 
  • Allows to group download data
  • Allows to control status of data download using a dedicated GUI (particularly useful when downloading large files)
  • Allows to download files in a physical location different from the client from which the user is currently accessing the web client
  • Permits to create specific plug-ins that are capable of aggregating data in a manner which is specific of a given data type (e.g. aggregating Sentinel-2 granules)
 
Basically, the software works as manager of downloads of products Operator already chose from other external clients and let him to decide, also from different locations, when e where start or stop it. In addition, Operator will also be able to execute different operations on downloads, like pause some of downloads, cancel  other ones and so on.
