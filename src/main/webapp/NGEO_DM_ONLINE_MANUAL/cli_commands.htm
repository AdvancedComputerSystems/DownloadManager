CLI COMMANDS

The complete list of DownloadManagerCLI commands and their description is reported below.
All command shall be preceded by a ‘-‘ sign in the CLI prompt.
 
 
Here is a more detailed description of single CLI commands:
 
Commands
Functions
listAll
Displays the downloads list of current downloads. The following information is reported:
·     Filename: URL of the file being downloaded
·     Status : can be any of the following:
o     DOWNLOADING: currently being downloaded
o     COMPLETED: download completed
o     PAUSED: paused (it can be resumed at later stage)
o     CANCELLED: cancelled (it cannot be resumed)
o     NOT_STARTED: in the queue, waiting to be downloaded
o     ERROR: an error has occurred during the download
·     FileSize: total size of the file in bytes
·     Progress: progress in percentage
·     FileSource: it is the name of the DAR from which the URL being downloaded was extracted, or set to MANUAL_DOWNLOAD is not associated to any DAR (e.g. activated with the –download command)
·     DownloadId: internal GID. To be used for all commands needing it (see below)
Network: network type, e.g. HTTP GET, BitTorrent, etc.
 
download [URI]
starts URI download.
info[URI_id]
Show information related to non-completed products with the specified <GID>. This command gives information about product status (if active, paused or cancelled), about product size and on downloaded chunk. This is the list of detailed information (please refer to for more technical details):
  • pieceLength: length of the individual download pieces in bytes
  • bitField: a bit field used to represent the total progress of the download
  • completedLength: length currently downloaded in bytes
    status: status of the download (e.g. active, error, paused)
pause  [URI_id]
pause the product with the specified identifier. The product identifier is given back by the download command. The command has sense only for active downloads. The product download can be resumed through the resume command.
resume [URI_id]
resume the product with the specified identifier. The command can only be applied for paused downloads. If the user tries to apply it to a download not in status paused, an error Is returned.
remove [URI_id]
cancel the product with the specified identifier. The product is permanently removed from download list and will be put into removed status.
changePriority [URI_id]
Put the product with the specified identifier at the top of the queue of waiting downloads. The command only applies for waiting downloads i.e. products added to the download list but still waiting to be downloaded.
addDAR [DAR URL]     
Read DAR specified in the DAR URL and starts downloading all contained Product URIs. Products are also added into the ngEO-D database and can be monitored and managed through the GUI.
cancelDAR [DAR URL]
Cancel downloading all Product URIs contained in the specified DAR. Products are permanently removed ngEO-D download list and will be put into removed status. The download of products listed in the given DAR URL can not be resumed.
pauseDAR [DAR URL]
Pause downloading all Product URIs contained in the specified DAR that are in active status (i.e. whose download isn’t finished yet). The download of products listed in the given DAR URL can be resumed through the resume command.
resumeDAR [DAR URL]
Resume downloading all Product URIs contained in the specified DAR that are in pause status (i.e. whose download was paused through the pause command).
showConfig
gives information about download manager configuration. In particular the commands gives information on central web server URL, username and password used to get DAR list and the directory where downloaded files are stored. To change the download manager configuration it is possible to act on configuration file or to use the GUI.
 
testConfig
Performs a test on the current configuration file, and verifies if the config file is well formed (e.g. returns an error if the config file is missing a mandatory parameter).
setConfig [key][value]
Allows to change the configuration file, setting the parameters identified by [key] to the value specified in [value].
help
Displays help on ngEO-D commands (See 1st image on the top).
bye
Exits from the CLI.
